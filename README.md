Чтобы запустить проект, тебе нужно сначала создать пакет exam_robot командой cd ~/ros2_ws/src и ros2 pkg create exam_robot --build-type ament_python --dependencies rclpy std_msgs geometry_msgs, затем скопировать все файлы узлов battery_node.py, distance_sensor_node.py, robot_status_node.py и robot_controller_node.py в папку exam_robot/exam_robot/, потом создать папку urdf в exam_robot/ и положить туда файл my_robot.urdf с моделью робота, также создать папку launch в exam_robot/ и положить туда файл robot_launch.py, после этого отредактировать setup.py добавив в data_files пути к urdf и launch через os.path.join и glob, а в entry_points прописать console_scripts для всех четырех узлов, затем отредактировать package.xml добавив зависимости robot_state_publisher и urdf, а также exec_depend ros2launch, далее собрать пакет командами cd ~/ros2_ws, colcon build --packages-select exam_robot и source install/setup.bash, после этого запустить всю систему одной командой ros2 launch exam_robot robot_launch.py, потом в отдельном терминале проверить что все узлы работают командой ros2 node list где должны быть battery_node, distance_sensor, status_display, robot_controller и robot_state_publisher, также проверить топики ros2 topic list где должны быть /battery_level, /distance, /robot_status, /cmd_vel, /tf и /robot_description, далее можно посмотреть статус робота через ros2 topic echo /robot_status и команды скорости через ros2 topic echo /cmd_vel, для тестирования разных режимов вручную публиковать значения батареи и расстояния например для нормального режима ros2 topic pub /battery_level std_msgs/Float32 "data: 80.0" --once и ros2 topic pub /distance std_msgs/Float32 "data: 2.0" --once, для низкого заряда ros2 topic pub /battery_level std_msgs/Float32 "data: 15.0" --once, для близкого препятствия ros2 topic pub /distance std_msgs/Float32 "data: 0.8" --once, для критического режима ros2 topic pub /battery_level std_msgs/Float32 "data: 5.0" --once или ros2 topic pub /distance std_msgs/Float32 "data: 0.5" --once, и если хочешь увидеть модель робота визуально запусти rviz2 командой ros2 run rviz2 rviz2 и добавь дисплей RobotModel выбрав топик /robot_description а также дисплей TF чтобы увидеть оси координат.
